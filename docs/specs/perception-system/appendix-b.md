## B.1 Field of View Verification (FOV Tests)

**Constants used (§3.10):**
- `BASE_FOV_ANGLE` = 160°
- `MAX_FOV_BONUS_ANGLE` = 10°
- `MAX_FOV_PRESSURE_REDUCTION` = 30°
- `MIN_FOV_ANGLE` = 120°

**FoV formula (Section 3 v1.2 authoritative):**
```
bonus = (Decisions / 20.0f) × MAX_FOV_BONUS_ANGLE
PressureReduction = PressureScalar × MAX_FOV_PRESSURE_REDUCTION
EffectiveFoV = Max(BASE_FOV_ANGLE + bonus − PressureReduction, MIN_FOV_ANGLE)
EffectiveFoV_HalfAngle = EffectiveFoV / 2.0f
```

---

### B.1.1 FOV-001: Centre-Facing Entity — Always Visible

**Test ref:** §5.2 FOV-001  
**Setup:** Decisions=10, PressureScalar=0. Target at 0° angular offset.

```
Step 1 — Decisions bonus:
  bonus = (10/20) × 10° = 0.5 × 10° = 5.0°

Step 2 — Pressure reduction (PressureScalar=0):
  reduction = 0 × 30° = 0°

Step 3 — EffectiveFoV:
  EffectiveFoV = 160° + 5.0° − 0° = 165.0°
  Floor check: 165° > 120° → no clamp
  EffectiveFoV_HalfAngle = 82.5°

Step 4 — Visibility test:
  angularSeparation = 0° < 82.5° → inFoV = true ✓
```

**Confirmed:** IsVisible = true. ✓

### B.1.2 FOV-004: Decisions Range — Correct Expected Values

**Test ref:** §5.2 FOV-004  
**Setup:** PressureScalar=0. Decisions=1 vs Decisions=20.

```

  bonus = (1/20) × 10° = 0.05 × 10° = 0.5°

Decisions=20 (maximum):
  bonus = (20/20) × 10° = 1.0 × 10° = 10.0°
  EffectiveFoV = 160° + 10.0° = 170.0°

Difference: 170.0° − 160.5° = 9.5°
```

**Confirmed expected values for Section 5 FOV-004:**

The bonus range spans 0.5°–10.0° across Decisions [1,20] because the D/20 formula gives
D=1 a non-zero 0.5° baseline. The total bonus range is MAX_FOV_BONUS_ANGLE=10° minus the
baseline, yielding a measurable 9.5° spread across the full attribute range.

## B.3 Recognition Latency Verification (LR Tests)

**Constants used:**
- `L_MAX` = 5 ticks (500ms)
- `L_MIN` = 1 tick (100ms)
- `CONFIRMATION_EXPIRY_TICKS` = 1 tick
- `PERIPHERAL_ARC_INNER_BOUND` = 40°
- **Rounding convention: Mathf.FloorToInt() — authoritative per §3.3.2**

**Base formula:**
```
L_rec_base_float = 5 − ((Decisions − 1) / 19.0f) × 4
L_rec_base_ticks = Mathf.FloorToInt(L_rec_base_float)
```

---

**Test ref:** §5.4 LR-001, LR-002

```

```

**Confirmed:** Endpoints produce exact integers. ✓

---

### B.3.2 LR-003: Mid-Range — Floor Rounding Is Deterministic

**Test ref:** §5.4 LR-003  
**Setup:** Decisions=10.

```
L_rec_float = 5 − (9/19) × 4
            = 5 − 0.473684... × 4
            = 5 − 1.894736...
            = 3.105263...

FloorToInt(3.105...) = 3 ticks
```

**Confirmed:** L_rec = **3 ticks** at Decisions=10. Floor rounding is deterministic — no
±1 tolerance applies. An implementation producing 4 ticks at this input is using ceiling
or round() and must be corrected. ✓

### B.3.3 LR-004: Half-Turn Bonus Applied in Peripheral Arc

**Test ref:** §5.4 LR-004  
**Setup:** Decisions=10 (L_rec=3 ticks). Target at 55°. IsHalfTurned=true.

```
Peripheral arc check: |55°| >= 40° ✓ AND |55°| <= 80° ✓ → in peripheral arc

Half-turn bonus:
  FloorToInt(3 × 0.85) = FloorToInt(2.55) = 2 ticks
  Floor check: 2 >= L_MIN=1 ✓ → no clamp needed
  L_rec = 2 ticks
```

**Confirmed:** L_rec reduced from 3 → 2 ticks with half-turn bonus. ✓

---

### B.3.4 LR-006: Additive Noise Cannot Violate L_MIN

**Test ref:** §5.4 LR-006  
**Setup:** Decisions=20, noise=+1 (maximum additive noise case).

```
L_rec_base = 1 tick = L_MIN
noise = +1 tick
L_rec_final = Min(1 + 1, L_MAX=5) = Min(2, 5) = 2 ticks

2 >= L_MIN ✓ — floor not violated
```

**Proof summary:** noise ∈ {0,1}, always additive. L_rec_adjusted ≥ L_MIN always.
Therefore L_rec_final = L_rec_adjusted + noise ≥ L_MIN. No secondary clamp required. ✓
